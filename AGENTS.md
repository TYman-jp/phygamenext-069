# AGENTS.md

このリポジトリは、Node.js の静的ファイルサーバーで動作する物理シミュレーション Web アプリ「PhyGame Next」です。主題は光の屈折で、Canvas 上に入射光・屈折光・境界面・法線を描画し、スネルの法則に基づいて屈折角や全反射を計算します。

## プロジェクトの目的

- Node.js を用いて、物理シミュレーションができる Web システムを開発する。
- 現在の中心機能は「光の屈折シミュレーション」。
- ユーザーが入射角、媒質 1 の屈折率 `n1`、媒質 2 の屈折率 `n2` を設定し、光線の経路と屈折角を確認できるようにする。
- 実行日時、入力パラメータ、実行回数をログとして取得・保存・CSV 出力できるようにする。
- 屈折後の光を的に当てるためのパラメータを答える「問題・回答機能」を備える。問題生成、回答判定、答え表示は `quiz.js` が担当する。

## 技術スタック

- 実行環境: Node.js `>=14.0.0`
- サーバー: Node.js 標準モジュールのみを使った静的ファイルサーバー
- フロントエンド: HTML / CSS / プレーン JavaScript
- 描画: HTML Canvas 2D API
- 永続化: ブラウザの `localStorage`
- 依存パッケージ: 現状なし

## 起動と検証

- 起動: `node server.js`
- ポート指定: `node server.js 8080`
- 既定 URL: `http://localhost:3000`
- npm 経由: `npm start` または `npm run dev`
- JS 構文確認:
  - `node --check server.js`
  - `node --check app.js`
  - `node --check simulation.js`
  - `node --check logger.js`

変更後は、少なくとも編集した JavaScript ファイルに対して `node --check` を実行してください。UI や Canvas 描画に関わる変更では、ブラウザで `http://localhost:3000` を開いて、スライダー操作、実行ボタン、ログ追加、CSV 出力、全反射表示が破綻していないことを確認してください。

## ファイル構成と責務

- `index.html`
  - アプリの HTML エントリーポイント。
  - Canvas、パラメータ操作パネル、プリセット、数式表示、実行ボタン、ログテーブルを配置する。
  - スクリプト読み込み順は `simulation.js`、`logger.js`、`app.js` が基本。

- `style.css`
  - ダークな光学実験室テーマのスタイル。
  - 既存の主要カラーは背景 `#0A0E1A`、入射光 `#00D4FF`、屈折光 `#FF6B35`、パネル `#1A2744`、テキスト `#F0F4FF`。
  - フォント設計は見出し `Space Grotesk`、UI `Inter`、数値・ログ `JetBrains Mono`。
  - 操作パネルとログパネルは横並び、狭い画面では縦並びにする。

- `simulation.js`
  - `RefractionSimulation` クラスを定義する。
  - スネルの法則による屈折角計算、全反射判定、Canvas 描画、アニメーションを担当する。
  - `calcRefraction(theta1_deg, n1, n2)` は屈折角を度数で返し、全反射の場合は `null` を返す。
  - `compute(theta1, n1, n2)` は `{ theta2, isTIR }` を返す。
  - `run(params)` はアニメーション付きで描画し、`draw(params, progress)` は現在状態を描画する。

- `logger.js`
  - `SimulationLogger` クラスを定義する。
  - 実行ログを `localStorage` の `phygame_logs` に保存する。
  - ログ項目は実行 ID、日時、`n1`、`n2`、入射角、屈折角、全反射かどうか。
  - CSV 出力では日本語ヘッダーと BOM 付与を想定する。

- `quiz.js`
  - `RefractionQuiz` クラスを定義する。
  - 的の位置、媒質の組み合わせ、正解の入射角を含む問題を生成する。
  - 現在のスライダー値を回答として評価し、屈折率、入射角、的への命中距離を使って正誤判定する。
  - `revealAnswer()` で正解の `n1`、`n2`、入射角、屈折角を表示する。

- `app.js`
  - DOM 要素の取得、スライダーイベント、プリセット、実行ボタン、ログ表示、CSV エクスポートを担当するアプリケーションコントローラー。
  - `RefractionSimulation` と `SimulationLogger` を生成して連携する。
  - スライダー変更時はリアルタイムで数式と Canvas プレビューを更新する。
  - 実行ボタン押下時はアニメーション実行、ログ追加、全反射警告更新を行う。

- `server.js`
  - Node.js 標準 `http` / `fs` / `path` による静的ファイルサーバー。
  - ルート `/` は `index.html` を返す。
  - MIME type と `Cache-Control: no-cache` を設定する。
  - パストラバーサル対策として `__dirname` 配下のファイルだけ返す。

- `package.json`
  - プロジェクト定義と npm scripts。
  - 外部依存は追加されていないため、依存導入が必要な場合は目的と影響を明確にしてから行う。

## 物理シミュレーションの仕様

- 入射角は法線からの角度として扱う。
- スネルの法則:

```text
n1 * sin(theta1) = n2 * sin(theta2)
```

- 実装では `theta1` と `theta2` は UI 表示上は度数法、計算時はラジアンへ変換する。
- `sin(theta2) = (n1 / n2) * sin(theta1)` が `1` を超える場合は全反射として扱う。
- 全反射時は屈折角を表示せず、警告・ログ・Canvas 表示で全反射と分かるようにする。
- 角度、屈折率、屈折角の丸め表示は既存 UI の粒度に合わせる。

## ログ仕様

- 実行ボタン押下ごとにログを 1 件追加する。
- ログは最新順に表示する。
- 保存件数は `localStorage` 側で最大 100 件を想定する。
- 表示は最新 50 件程度に抑える設計になっている。
- CSV ファイル名は `phygame_log_YYYYMMDD_HHMM.csv` 形式を維持する。
- CSV の文字化け対策として BOM `\uFEFF` を付与する。

## クイズ機能の仕様

`ai.txt` には、光の屈折先に的を用意し、その的に光を当てるためにどのパラメータをどの数値にすればよいか答える機能を追加したい、という要望が記録されています。現在は `quiz.js`、`index.html` のクイズパネル、`app.js` のイベント連携、`simulation.js` の的描画として実装されています。

現在の設計:

- `quiz.js` で問題生成・正解計算・回答判定を分離する。
- 問題は `n1`、`n2`、的の位置を生成し、的の位置から逆算して正解の入射角を求める。
- ユーザーは入射角スライダーなどを調整して実行し、光線が的の許容範囲に入れば正解とする。
- `simulation.js` には `setTarget()` / `clearTarget()` / `_drawTarget()` があり、Canvas 上に黄色い的を描画する。
- `index.html` にはクイズパネルがあり、問題生成、回答判定、答え表示のボタンを持つ。
- `app.js` では既存の `RefractionSimulation` インスタンスを `RefractionQuiz` に渡して再利用する。
- 現時点ではクイズ回答は通常ログには保存していない。ログ統合を行う場合は、通常実行ログとクイズ回答ログの区別がつく項目を追加する。

## UI / デザイン方針

- 暗色の実験室テーマを維持する。
- 入射光はシアン、屈折光はオレンジ、全反射は警告色として扱う。
- Canvas を主役にし、パラメータ操作とログは補助パネルとして配置する。
- 物理量やログの数値には等幅フォントを使う。
- ボタンやスライダーの操作後に、Canvas、数式、角度バッジ、警告、ログ件数が同期して更新されることを重視する。
- レスポンシブ表示では、操作パネルとログパネルが重ならないようにする。

## 実装時の注意

- 既存コードはプレーン JavaScript のクラスと即時関数で構成されている。ビルドツールやフレームワークを追加しない限り、この構成に合わせる。
- `app.js` は DOM 要素 ID に強く依存している。HTML の ID を変更する場合は、対応する JS 参照も必ず更新する。
- `simulation.js` の角度計算と Canvas 座標系を混同しない。角度は法線基準、Canvas は右向きが `+x`、下向きが `+y`。
- `localStorage` のキー `phygame_logs` を変更すると既存ログが復元できなくなるため、変更する場合は移行処理を検討する。
- サーバーは標準モジュールだけで動く設計なので、単純な静的配信のために Express などを追加しない。
- 既存の表示文字列には文字化けしている箇所がある。ユーザー向けテキストを触る場合は、UTF-8 の正しい日本語に直し、ブラウザ表示で確認する。
- 日本語テキストを編集したファイルは UTF-8 として保存する。

## 変更後チェックリスト

- `node --check` が通る。
- `node server.js` でサーバーが起動する。
- `http://localhost:3000` で画面が表示される。
- 入射角、`n1`、`n2` のスライダー変更で Canvas と数式が更新される。
- プリセットボタンで値が反映される。
- 実行ボタンでログが追加される。
- 全反射条件で警告とログが正しく表示される。
- 問題生成で Canvas に黄色い的が表示される。
- 回答判定で正解・不正解のメッセージが表示される。
- 答え表示で `n1`、`n2`、入射角、屈折角が表示される。
- ログクリアと CSV エクスポートが動く。
- UI 文字列を変更した場合、文字化けやレイアウト崩れがない。
