# AGENTS.md

このリポジトリは、Node.js の静的ファイルサーバーで動作する物理シミュレーション Web アプリ「PhyGame Next」です。現在は **光の屈折シミュレーション** と **水面波シミュレーション** の 2 つの実験モードを搭載しています。ヘッダーのタブで切り替えて使用します。

## プロジェクトの目的

- Node.js を用いて、物理シミュレーションができる Web システムを開発する。
- **光の屈折モード**: ユーザーが入射角、媒質 1 の屈折率 `n1`、媒質 2 の屈折率 `n2` を設定し、光線の経路と屈折角を確認できるようにする。
- **水面波モード**: 振幅・伝播速度・周波数・液体の粘度（減衰）を設定し、波の伝播・干渉・回折をリアルタイムでシミュレーションできるようにする。障害物の手描き・消去とプリセット（スリット・二重スリット・円柱）を用意する。
- 実行日時、入力パラメータ、実行回数をログとして取得・保存・CSV 出力できるようにする。ログは「通常（屈折）」「クイズ」「水面波」の 3 タイプを区別して保存する。
- 屈折後の光を的に当てるためのパラメータを答える「問題・回答機能」を備える。問題生成、回答判定、答え表示は `quiz.js` が担当する。クイズ回答もログに記録される。

## 技術スタック

- 実行環境: Node.js `>=14.0.0`
- サーバー: Node.js 標準モジュールのみを使った静的ファイルサーバー
- フロントエンド: HTML / CSS / プレーン JavaScript
- 描画: HTML Canvas 2D API（屈折は通常描画、水面波は `ImageData` + オフスクリーンキャンバスによるピクセル描画）
- 永続化: ブラウザの `localStorage`
- 依存パッケージ: 現状なし

## 起動と検証

- 起動: `node server.js`
- ポート指定: `node server.js 8080`
- 既定 URL: `http://localhost:3000`
- npm 経由: `npm start` または `npm run dev`
- JS 構文確認:
  - `node --check server.js`
  - `node --check app.js`
  - `node --check simulation.js`
  - `node --check wave_simulation.js`
  - `node --check logger.js`
  - `node --check quiz.js`

変更後は、少なくとも編集した JavaScript ファイルに対して `node --check` を実行してください。UI や Canvas 描画に関わる変更では、ブラウザで `http://localhost:3000` を開いて、変更後チェックリストに沿って動作確認を行ってください。

## ファイル構成と責務

- `index.html`
  - アプリの HTML エントリーポイント。
  - ヘッダーにモード切り替えタブ（「光の屈折」「水面波」）を配置する。
  - Canvas、屈折用パラメータパネル（`refractionControls`）、水面波用パラメータパネル（`waveControls`）、実行ボタン、クイズパネル（`quizPanel`）、ログテーブルを配置する。
  - スクリプト読み込み順は `simulation.js`、`wave_simulation.js`、`logger.js`、`quiz.js`、`app.js`。

- `style.css`
  - ダークな光学実験室テーマのスタイル。
  - 主要カラーは背景 `#0A0E1A`、入射光 `#00D4FF`、屈折光 `#FF6B35`、パネル `#1A2744`、テキスト `#F0F4FF`。
  - フォント設計は見出し `Space Grotesk`、UI `Inter`、数値・ログ `JetBrains Mono`。
  - モードタブ（`.mode-tabs`、`.tab-btn`）、障害物ペン/消しゴムツールボタン（`.tool-btn`）のスタイルを含む。
  - `body.mode-wave` クラス付与時は、クイズパネルを非表示にし、グリッドを 2 カラム（コントロール・ログ）に変更する。
  - ログ行のタイプ・結果バッジ用クラス: `.log-type-quiz`（オレンジ）、`.log-type-normal`（グレー）、`.result-correct`（緑）、`.result-miss`（黄/警告色）。
  - レスポンシブ表示では全パネルを縦並びにする。

- `simulation.js`
  - `RefractionSimulation` クラスを定義する。
  - スネルの法則による屈折角計算、全反射判定、Canvas 描画、アニメーションを担当する。
  - `calcRefraction(theta1_deg, n1, n2)` は屈折角を度数で返し、全反射の場合は `null` を返す。
  - `compute(theta1, n1, n2)` は `{ theta2, isTIR }` を返す。
  - `run(params)` はアニメーション付きで描画し、`draw(params, progress)` は現在状態を描画する。
  - `setTarget()` / `clearTarget()` / `_drawTarget()` により、クイズ用の黄色い的を Canvas 上に表示する。

- `wave_simulation.js`
  - `WaveSimulation` クラスを定義する。
  - 水面波モードの物理演算と Canvas 描画を担当する。
  - 2 次元離散波動方程式（差分法）を毎フレーム解き、波の伝播・反射・回折をリアルタイムでシミュレートする。
  - グリッドスケールは `scale = 4`（1 グリッド = 4×4 ピクセル）で計算負荷を抑制し、オフスクリーンキャンバスの `ImageData` を用いてピクセル単位で高速描画する。
  - 波源（左中央に固定）から正弦波を継続的に励起し、液体の粘度に対応する減衰係数 `d` と境界吸収係数を掛けることで波を減衰させる。
  - `setParams(params)` で振幅・速度・周波数・粘度を随時更新できる。
  - `addObstacle(canvasX, canvasY, radius)` / `removeObstacle(canvasX, canvasY, radius)` でキャンバス座標からグリッド座標に変換して障害物配列を操作する。
  - `setPreset(preset)` で `'slit'`（単一スリット）・`'double-slit'`（二重スリット）・`'pillar'`（円柱）・`'clear'`（全消去）を設定できる。
  - `hasObstacles()` は障害物の有無を真偽値で返す（ログ記録用）。
  - `run()` / `stop()` でアニメーションループを開始・停止する。
  - 波源の位置（左中央）は障害物で上書きされない。

- `logger.js`
  - `SimulationLogger` クラスを定義する。
  - 実行ログを `localStorage` の `phygame_logs` に保存する。
  - ログエントリには実行 ID、日時、`n1`（または振幅）、`n2`（または速度）、`theta1`（または周波数）、`theta2`（または粘度）、`isTIR`、`type`、`quizStatus`、`fixedOk`、`hasObstacles` を含む。
  - `type` は `'normal'`（屈折通常実行）、`'quiz'`（クイズ回答）、`'wave'`（水面波実行）の 3 種類。
  - `add(params, result)` は `params` の各フィールドを優先的にマッピングし、なければ `amplitude` / `speed` / `frequency` / `viscosity` を代入する。
  - CSV 出力では列ヘッダーを `n1/振幅`・`n2/速さ`・`入射角/周波数`・`屈折角/粘度` と表記し、タイプ別に結果列を切り替える。
  - CSV ファイルには BOM `\uFEFF` を付与して日本語の文字化けを防ぐ。

- `quiz.js`
  - `RefractionQuiz` クラスを定義する。
  - 的の位置、媒質の組み合わせ、固定条件、答えの一例を含む問題を生成する。
  - 出題タイプは、自由調整、入射角固定、`n1` 固定、`n2` 固定、媒質固定を混ぜる。
  - 回答判定では、保存済みの正答値との一致ではなく、現在のスライダー値で計算した屈折光が的の円内を通るかどうかで正解を判定する。
  - 固定条件がある問題では、固定値から外れている場合は不正解とし、固定値を合わせるようメッセージを返す。
  - `revealAnswer()` は「答えの一例」として `n1`、`n2`、入射角、屈折角を表示する。これは判定用の唯一解ではない。

- `app.js`
  - `RefractionSimulation`、`WaveSimulation`、`SimulationLogger`、`RefractionQuiz` を生成して連携するアプリケーションコントローラー（即時関数）。
  - `currentMode` 変数（`'refraction'` / `'wave'`）でアクティブなモードを管理する。
  - タブクリック時に `body` クラス（`mode-refraction` / `mode-wave`）を付け替え、対応するパラメータパネルの表示を切り替え、`waveSim.run()` / `waveSim.stop()` を制御する。
  - 屈折モードのスライダー変更時はリアルタイムで数式と Canvas プレビューを更新する。
  - 水面波モードのスライダー変更時は `waveSim.setParams()` でリアルタイムに反映する。
  - キャンバス上のマウス・タッチイベントを検知し、アクティブツール（ペン/消しゴム）で `waveSim.addObstacle()` / `waveSim.removeObstacle()` を呼び出す。波源の位置はドラッグで上書きされない。
  - 実行ボタン押下時は、屈折モードならアニメーション実行・ログ追加、水面波モードなら `waveSim.clear()` で波をリセット・ログ追加を行う。
  - クイズ問題生成時は、固定条件がある場合に該当スライダーを固定値へ同期する。クイズ回答判定時はログに `type: 'quiz'` として記録する。
  - `addLogRow(entry)` は `type` に応じて 8 列のテーブル行 HTML を生成する（水面波行では `theta1` を `Hz` 表記、`theta2` を粘度値として表示する）。

- `server.js`
  - Node.js 標準 `http` / `fs` / `path` による静的ファイルサーバー。
  - ルート `/` は `index.html` を返す。
  - MIME type と `Cache-Control: no-cache` を設定する。
  - パストラバーサル対策として `__dirname` 配下のファイルだけ返す。

- `package.json`
  - プロジェクト定義と npm scripts。
  - 外部依存は追加されていないため、依存導入が必要な場合は目的と影響を明確にしてから行う。

## 物理シミュレーションの仕様

### 光の屈折

- 入射角は法線からの角度として扱う。
- スネルの法則:

```text
n1 * sin(theta1) = n2 * sin(theta2)
```

- `theta1` と `theta2` は UI 表示上は度数法、計算時はラジアンへ変換する。
- `sin(theta2) = (n1 / n2) * sin(theta1)` が `1` を超える場合は全反射として扱う。
- 全反射時は屈折角を表示せず、警告・ログ・Canvas 表示で全反射と分かるようにする。

### 水面波

- 2 次元離散波動方程式（差分法）:

```text
u[i][j][t+1] = (2*u[i][j][t] - u[i][j][t-1] + c²*(隣接4セルの和 - 4*u[i][j][t])) * d * boundary[i][j]
```

- `c`: 伝播速度（ユーザー設定、`0.10 〜 1.00`）
- `d`: 減衰係数（粘度に対応、`0.950 〜 0.998`。値が小さいほど高粘度で早く減衰する）
- `boundary[i][j]`: 外周 6 グリッド内で `0.85〜1.0` に線形補間した吸収係数（反射抑制）
- 障害物セルは常に `0` に固定（固定端境界）
- 安定性確保のため `c_sq = c² * 0.5` を使用する

## ログ仕様

- 実行ボタン押下またはクイズ回答判定ごとにログを 1 件追加する。
- ログは最新順に表示する。
- 保存件数は `localStorage` 側で最大 100 件を想定する。
- 表示は最新 50 件程度に抑える設計になっている。
- CSV ファイル名は `phygame_log_YYYYMMDD_HHMM.csv` 形式を維持する。
- CSV の文字化け対策として BOM `\uFEFF` を付与する。
- ログのタイプは `type` フィールドで識別する:
  - `'normal'`: 屈折シミュレーション通常実行
  - `'quiz'`: クイズ回答判定（`quizStatus`・`fixedOk` を含む）
  - `'wave'`: 水面波シミュレーション実行（`hasObstacles` を含む）

## クイズ機能の仕様

- `quiz.js` で問題生成・回答判定・答えの一例の表示を分離する。
- 問題は `n1`、`n2`、入射角の一例から的の位置を生成する。
- 出題時には、自由調整問題に加えて、入射角、`n1`、`n2`、媒質のいずれかを固定した問題も生成する。
- 固定条件がある問題では、`app.js` が該当スライダーを固定値に同期する。
- ユーザーは固定条件を守りつつ、残りのスライダーを調整して回答する。
- 回答判定では、あらかじめ用意した唯一の正答と比較しない。現在の `theta1`、`n1`、`n2` から屈折光を計算し、その光線が的の円内を通っていれば正解とする。
- 全反射の場合は、屈折光が的まで進まないため不正解とする。
- 固定条件から外れている場合は、命中判定の前に不正解とし、どの固定値を合わせるべきかを表示する。
- `revealAnswer()` は正答候補の一例を表示するための機能であり、正誤判定の基準ではない。
- クイズ回答判定時は、通常ログと区別するため `type: 'quiz'`、`quizStatus`（`'correct'` / `'miss'`）、`fixedOk` を付与してログに保存する。

## UI / デザイン方針

- 暗色の実験室テーマを維持する。
- 入射光はシアン、屈折光はオレンジ、全反射は警告色（黄）として扱う。
- Canvas を主役にし、パラメータ操作、クイズ、ログは補助パネルとして配置する。
- 物理量やログの数値には等幅フォントを使う。
- ボタンやスライダーの操作後に、Canvas、数式、角度バッジ、警告、ログ件数、クイズ状態が同期して更新されることを重視する。
- 水面波モード時はクイズパネルを非表示にし、レイアウトを 2 カラムに変更する（CSS の `body.mode-wave` クラスで制御）。
- レスポンシブ表示では、全パネルが縦並びになる。

## 実装時の注意

- 既存コードはプレーン JavaScript のクラスと即時関数で構成されている。ビルドツールやフレームワークを追加しない限り、この構成に合わせる。
- `app.js` は DOM 要素 ID に強く依存している。HTML の ID を変更する場合は、対応する JS 参照も必ず更新する。
- `simulation.js` の角度計算と Canvas 座標系を混同しない。角度は法線基準、Canvas は右向きが `+x`、下向きが `+y`。
- `wave_simulation.js` のグリッド座標とキャンバス座標は `scale` 倍の関係にある（グリッド座標 × `scale` = キャンバス座標）。
- `wave_simulation.js` の差分法パラメータ `c_sq = c * c * 0.5` は数値安定性のための設定値であり、変更する場合は発散しないことを確認する。
- クイズ判定は `quiz.js` の光線と的の距離計算に依存する。正答候補との数値比較を判定条件として復活させない。
- 固定条件つきクイズでは、固定値から外れている回答を正解にしない。
- `localStorage` のキー `phygame_logs` を変更すると既存ログが復元できなくなるため、変更する場合は移行処理を検討する。
- サーバーは標準モジュールだけで動く設計なので、単純な静的配信のために Express などを追加しない。
- 日本語テキストを編集したファイルは UTF-8 として保存する。

## 変更後チェックリスト

### 共通
- `node --check` が通る。
- `node server.js` でサーバーが起動する。
- `http://localhost:3000` で画面が表示される。
- ログクリアと CSV エクスポートが動く。
- UI 文字列を変更した場合、文字化けやレイアウト崩れがない。

### 光の屈折モード
- 「光の屈折」タブをクリックするとパラメータパネルが切り替わり、水面波のアニメーションが停止する。
- 入射角、`n1`、`n2` のスライダー変更で Canvas と数式が更新される。
- プリセットボタンで値が反映される。
- 実行ボタンでアニメーションが再生され、`通常` タイプのログが追加される。
- 全反射条件で警告とログが正しく表示される。
- 問題生成で Canvas に黄色い的が表示される。
- 問題生成で、自由調整問題と固定条件つき問題が出題される。
- 固定条件つき問題では、該当スライダーが固定値に同期される。
- 回答判定で、的に光が当たっていれば正解、外れていれば不正解のメッセージが表示され、`クイズ` タイプのログが追加される。
- 固定条件から外れた回答では、固定値を合わせるようメッセージが表示される。
- 全反射になる回答では不正解として表示される。
- 答え表示で `n1`、`n2`、入射角、屈折角の一例が表示される。

### 水面波モード
- 「水面波」タブをクリックするとパラメータパネルが切り替わり、クイズパネルが非表示になり、水面波アニメーションが開始する。
- 振幅・速度・周波数・粘度のスライダー変更がリアルタイムで波の動きに反映される。
- 「障害物ペン」でキャンバスをドラッグすると障害物が描画される。波源の位置には障害物が描画されない。
- 「消しゴム」でドラッグすると障害物が消去される。
- 各プリセットボタン（単一スリット・二重スリット・円柱障害物・全消去）が正しく動作する。
- 実行ボタンで波がリセットされ再スタートし、`水面波` タイプのログが追加される。
- 障害物がある状態と障害物なしの状態でログの「結果」列の表示が切り替わる。
