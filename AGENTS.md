# AGENTS.md

このリポジトリは、物理シミュレーション Web アプリ「PhyGame Next」です。現在は **光の屈折シミュレーション**、**水面波シミュレーション**、および **ログ** の 3 つのタブモードを搭載しています。ヘッダーのタブで切り替えて使用します。

## プロジェクトの目的

- 物理シミュレーションができる Web システムを開発する。
- **光の屈折モード**: ユーザーが入射角、媒質 1 の屈折率 `n1`、媒質 2 の屈折率 `n2` を設定し、光線の経路と屈折角を確認できるようにする。
- **水面波モード**: 振幅・伝播速度・周波数・液体の粘度（減衰）を設定し、波の伝播・干渉・回折をリアルタイムでシミュレーションできるようにする。障害物の手描き・消去とプリセット（スリット・二重スリット・円柱）を用意する。
- **ログモード (新規)**: シミュレーションの実行履歴を閲覧・管理する専用タブ。光の屈折系ログ（通常・クイズ）と水面波ログを左右に並べて表示する。
- 実行日時、入力パラメータ、実行回数をログとして取得・保存・CSV 出力できるようにする。ログは「通常（屈折）」「クイズ」「水面波」の 3 タイプを区別して保存する。
- 屈折後の光を的に当てるためのパラメータを答える「問題・回答機能」を備える。問題生成、回答判定、答え表示は `quiz.js` が担当する。クイズ回答もログに記録される。

## 技術スタック

- 実行環境: 静的ファイル配信環境 (Netlify などを想定)
- サーバー: （ローカル動作用）Node.js 標準モジュールによる静的ファイルサーバー
- フロントエンド: HTML / CSS / JavaScript (ES Modules)
- 描画: HTML Canvas 2D API（屈折は通常描画、水面波は `ImageData` + オフスクリーンキャンバスによるピクセル描画）
- 永続化: Firebase Firestore (クラウドデータベース)。ローカルファイルやサーバー保存は行わず、ブラウザから直接 Firebase へ保存します。
- 依存パッケージ: Firebase SDK (CDN 経由)

## 起動と検証

- ローカル起動: `node server.js`
- ポート指定: `node server.js 8080`
- 既定 URL: `http://localhost:3000`
- デプロイ: GitHub と Netlify を連携し、本番公開。
- JS 構文確認:
  - `node --check server.js`
  - `node --check app.js`
  - `node --check simulation.js`
  - `node --check wave_simulation.js`
  - `node --check logger.js`
  - `node --check quiz.js`

変更後は、少なくとも編集した JavaScript ファイルに対して `node --check` を実行してください。UI や Canvas 描画に関わる変更では、ブラウザで `http://localhost:3000` を開いて、変更後チェックリストに沿って動作確認を行ってください。

## 物理シミュレーションの仕様

### 光の屈折
- スネルの法則: `n1 * sin(theta1) = n2 * sin(theta2)`
- 全反射時は屈折角を表示せず、警告・ログ・Canvas 表示で全反射と分かるようにする。

### 水面波
- 2 次元離散波動方程式（差分法）:
`u[i][j][t+1] = (2*u[i][j][t] - u[i][j][t-1] + c²*(隣接4セルの和 - 4*u[i][j][t])) * d * boundary[i][j]`
- 「枠を壁として扱う（反射）」チェックボックスで、境界条件を「固定端（壁反射）」と「開放端（吸収境界）」に切り替え可能。
- 波の描画色は強調されたカラースケール（`colorScale = amplitude * 0.35`）で視認性を高めている。

## ログ仕様

- 実行ボタン押下またはクイズ回答判定ごとにログを 1 件追加する。
- 初回アクセス時にユーザーの名前を入力させ（ブラウザに保存）、ログレコードに含める。
- ログは最新順に表示する。
- ログのタイプは「通常」と「クイズ」の2種類のみとする（水面波の実行も「通常」と表記）。
- 保存場所は Firebase Firestore (`refractionLogs`, `waveLogs` コレクション) とする。
- Firebase 設定が未入力の場合は一時的にオンメモリ(ダミー)で動作する。
- CSV ファイル名は `phygame_log_refraction_YYYYMMDD_HHMM.csv` 形式（BOM付き）。
- **ログタブは開発者専用**: ログタブをクリックするとパスワード入力モーダルが表示され、正しいパスワードを入力した場合のみログを閲覧できる。認証状態は `sessionStorage` に保持され、ブラウザを閉じるとリセットされる。デフォルトパスワードは `phygame-dev`。SHA-256 ハッシュで比較する。

## クイズ機能の仕様
- `quiz.js` で問題生成・回答判定・答えの一例の表示を分離する。
- クイズ回答判定時は、通常ログと区別するため `type: 'quiz'` として Firestore に保存する。

## UI / デザイン方針
- 暗色の実験室テーマを維持する。
- レスポンシブ表示では、全パネルが縦並びになる。
